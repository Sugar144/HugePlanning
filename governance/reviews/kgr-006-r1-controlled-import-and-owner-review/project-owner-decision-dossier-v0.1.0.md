## ER-012 boundary in plain language

Exact boundary: **Do not process sensitive, regulated, confidential, or actively harmful real data until the applicable boundary is established.**

Practically, repository analysis with preserved documents may continue, but a later workflow must not use real data in any of those categories merely because it is called a pilot or because tooling is available. The relevant legal/privacy/security boundary must first be established for the actual data, purpose, jurisdiction, provider and threat context. This prohibition is broader than the obsolete wording that blocked only a governed pilot.
