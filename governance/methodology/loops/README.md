# Governance Methodology Loops

This area contains bounded, versioned orchestration methods that connect independent governance roles. A loop specification controls routing, counters, provenance, and guard exhaustion; it is not constitutional authority, a review result, or proof that any run occurred.

- `kernel-design-closure/` — `GOV-LOOP-001`, Kernel Design Closure Loop `0.1.0`, active for initial trial.
