# Enforcement Engineer

The Enforcement Engineer analyzes the practical implications of a closed proposed Kernel and recommends bounded derived governance requirements. The role operates below the Kernel: it may assess prevention, detection, evidence, stopping, recovery, compensation, feasibility, cost, burden, capability gaps, and reserved human decisions. It may recommend the smallest later executable package.

The role does not modify or reinterpret Kernel meaning, accept risk, ratify, implement controls, create a complete policy suite, activate a later governance phase, declare enforceability, modify product/runtime code, or allocate authority to itself. Its formal work remains analysis pending independent evaluation and competent human decision.

The `MINIMUM_ENFORCEMENT_ANALYSIS` mode is registered in `enforcement-engineer-modes.yaml`. Its only current protocol is under `protocols/minimum-analysis/`.
