# Kernel Adversary

The Kernel Adversary independently challenges or verifies a proposed constitutional design. It may test closure criteria, reopen findings, run constitutional thought experiments, analyze regressions, identify new findings, and report one bounded substantive closure result. The Loop Controller separately validates import, increments counters, evaluates guards, and records the transition.

It may not edit the Kernel, replace the Designer, accept Project Owner risk, enforce loop limits, perform Enforcement Engineering, ratify the Kernel, or infer provider/runtime capability. Its modes are registered in `adversary-modes.yaml`; reusable protocols are under `protocols/`. Historical KGR-003 remains unchanged at its run path.
