# Model Routing Policy

Use deterministic scripts for parsing, hashing, inventory, comparison, graph checks, serialization, and schema validation. Use the least expensive model that can reliably satisfy the semantic contract. A future pass records the actual model description, availability, reasoning mode, cost fields, and independence relationship when observable; it does not infer capabilities from names.

| Work class | Initial model class | Escalation trigger |
|---|---|---|
| Repository inventory and extraction | deterministic first; economical bounded model for classification | unresolved semantic ambiguity or material contradiction |
| Architecture synthesis and strategy comparison | strongest available reasoning class justified by the contract | failed coherence/evidence review |
| External research | economical research-capable class plus primary-source verification | contested, high-impact, or rapidly changing claim |
| Schema/test generation | deterministic generator or economical coding model | integration novelty or repeated correction |
| Adversarial review | independent capable reasoning class | material risk or unresolved architecture boundary |
| Final synthesis | capable synthesis class | predecessor conflict or incomplete evidence |
| Independent evaluation | evaluator outside synthesis author's unilateral control | independence or competence cannot be established |

`Terra` and `Luna` were not observed as available model choices during scaffold creation. They receive no routing or inferred strengths. A future pass may include either only after availability, official/observed descriptions, cost characteristics, and small pilot evidence are preserved.

AI-first estimates use execution cycles (`1`, `2-3`, `4-6`, `MORE_THAN_6`, or `UNKNOWN_UNTIL_PROTOTYPE`), Owner effort, validation burden, novelty, correction uncertainty, integration risk, and maintenance burden. Calendar time is optional, secondary, and assumption-bound.
