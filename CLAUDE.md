# Freelance Methodology Invariants

- Git is the source of truth. Uncommitted chat context is never project truth.
- Information precedence is: confirmed evidence; approved requirements and
  decisions; structured contracts; explanatory documents; operational views;
  agent memory; unrecorded conversation.
- Never convert an inference, assumption, preference, or methodology default
  into a client fact.
- Use stable canonical IDs. Never renumber, reuse, or silently replace an ID.
- Agents draft and recommend. Humans approve every gate and irreversible choice.
- Client data belongs only in that client's repository. Methodology examples are
  fictitious and contain no client data.
- The methodology repository is read-only during every client session.
- Consult `knowledge/INDEX.md` before improvising methodology knowledge when that
  index exists at the active implementation stage.
- Validate structured artifacts against the locked schemas before declaring them
  complete.
- Approved artifacts change only through the defined change-control process;
  supersede history instead of rewriting it.
