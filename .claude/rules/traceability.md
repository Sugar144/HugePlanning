# Traceability

## Policy

Use canonical IDs from `id-and-status-conventions.md`. Every reference must
resolve to an existing canonical item. Preserve upstream links when deriving
documents, backlogs, tests, code, and releases. Jira keys are labels, never
replacements for canonical IDs.

## Why

Impact analysis and verification depend on an unbroken evidence-to-release
chain.

## Observable violation

A dangling reference, an un-ID'd normative statement, a Jira key used as the
only identity, a renamed/reused ID, or a derived artifact with no upstream link.
